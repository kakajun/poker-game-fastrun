# 阶段 1：游戏核心引擎（纯规则，无 AI）实施方案

## 目标与范围
- 搭建完整、可测的跑得快（三人版，45 张牌）规则引擎，作为后续 AI 环境与 API 的唯一“裁判”。
- 范围包含：发牌、出牌合法性判定、牌型识别与比较、轮转与首出、必压/顶大规则、结算与计分。
- 不包含：任何神经网络/学习逻辑、对外服务接口。

## 规则基准
- 基于 [mainRule.md](file:///f:/git/poker-game-fastrun/mainRule.md) 与逻辑补全：
  - **牌堆构成（45张）**：
    - 3, 4, 5, 6, 7, 8, 9, 10, J, Q：各 4 张（共 40 张）。
    - K：3 张（保留黑桃、红桃、梅花；去掉方块 K）。
    - A：1 张（保留黑桃 A）。
    - 2：1 张（保留红桃 2）。
    - *注：以上花色为默认配置，可修改但需固定。*
  - **牌型限制补全**：
    - **炸弹**：只有 3-Q 能组成炸弹（K/A/2 张数不足）。
    - **顺子/连对**：2 不参与（最大到 A）。
    - **三带一**：只能带单张，不能带对子（因为总张数限制）。
  - **首出**：持红桃 3 的玩家必须首轮包含红桃 3。
  - **必压**：能管必须管；顶大：下家报单时上家需出当前可出的最大单张。

## 核心模块设计
- Deck 与发牌
  - DeckBuilder：按配置生成 45 张牌；洗牌；均分发牌。
  - 校验：牌总数=45，发牌后每人 15 张且无重复。
- 牌型识别（HandType）
  - 类型枚举：Single/Pair/Triple/TripleWithSingle/Straight/DoubleSequence/Airplane/AirplaneWithWings/Bomb/Pass。
  - 识别器：给定一组牌 -> 返回 {type, length, maxRank, aux}。对顺子/连对/飞机需返回长度与最大点。
  - 合法性：各类型长度与点数的边界（顺子>=5，连对>=3 对，飞机>=2 连三张等）。
- 牌力比较（Comparator）
  - 同型同张数：按点数比较；炸弹通吃非炸弹；炸弹之间比点数。
  - 特例：三带一比较三张点数；飞机带翅膀比较飞机主体最大点。
- 合法动作生成（ActionGenerator）
  - 输入：当前玩家手牌、当前桌面牌（上一手）、局面上下文（首轮/是否报单等）。
  - 输出：该玩家所有合法动作集合（包含 Pass 与否由规则控制：必压时不可 Pass）。
  - 规则：
    - 首轮含红桃 3；若当前为跟牌轮，必须同型同张数或炸弹。
    - 必压：若存在能管的牌型，则不能出 Pass 或其他不合法型。
    - 报单顶大：当下家牌数=1，且轮到我出牌时，若出单则必须出当前可出的最大单。
- 局面与轮转（GameState）
  - 字段：玩家序列、当前出牌玩家 idx、当前台面牌、上一手出牌者、各自剩余张数、首轮标志、报单标志等。
  - 流程：起始牌手（红桃 3）-> 轮流出牌 -> 一轮结束条件（其余两家过或被迫过）-> 新轮首家为上一手出牌者。
- 结算与计分
  - 胜利判定：任一玩家出完牌即结束。
  - 计分：基础分按剩余张数；炸弹即时结算；春天结算逻辑与分值。

## 数据结构与接口草案（不编码，先约定）
- Card：{rank, suit}
- Hand：Card[]（始终保持排序：rank 优先，suit 次之）
- Play：{type, cards[], meta} // meta 包含长度、最大点、是否炸弹等
- GameState（只读快照接口）：
  - players: 3
  - hands: Hand[3]
  - current_player: int
  - last_play: Play | null
  - last_player: int | null
  - first_round: boolean
  - reported_single: boolean[3]
  - scores_temp: {bomb_delta, …}
- Engine API（内部调用）
  - init(config) -> GameState
  - deal(state) -> GameState
  - legal_actions(state, player) -> Play[]
  - step(state, play) -> {next_state, events[]} // events: 炸弹得分、轮次变化、胜负

## 测试用例清单（必须通过）
- 牌堆构建
  - 总数=45；各点数计数正确；可配置的唯一 A/2/K 生效。
- 首轮与红桃 3
  - 持红桃 3 的玩家先手；首轮必须包含红桃 3；若含红桃 3 的顺子/连对/三带一判定正确。
- 牌型识别
  - 单/对/三张/三带一/顺子（跨 10-J-Q-K-A 的禁止规则）/连对/飞机/飞机带翅膀/炸弹的正反例。
- 比较逻辑
  - 同型同张数比较正确；炸弹压制正确；炸弹之间比较正确。
- 合法动作生成
  - 跟牌同型同张数集合准确；必压时无 Pass；可炸弹压任意；报单顶大约束生效。
- 轮转与结束
  - 一轮结束归属；胜利判定；计分正确；炸弹分与春天分即时/终结结算正确。

## 风险与缓解
- 牌型组合爆炸：对 ActionGenerator 做缓存/剪枝；优先按型搜索。
- 规则歧义（唯一 A/2/K 花色）：以配置化解决；配套测试覆盖不同配置。

- M2：比较与合法动作生成通过；首轮/必压/顶大规则生效。
- M3：流程与计分通过；完成验收。
